## 막혔던 것

- SSE fetch URL 중복 문제 발견: `VITE_API_BASE_URL=/api/v1` + fetch에서 `/api/v1/...` 사용 → `/api/v1/api/v1/...`가 되어 MSW 미매칭. 단, 이것은 기존 CS에도 동일하게 존재하는 환경 설정 이슈로 FN-QNA 구현 자체의 문제는 아님. QNA는 CS와 완전히 동일한 패턴을 따름.

## 브라우저 테스트 결과

- pnpm build: 통과

- 1차 답변(AI 답변 내용) 첫 메시지로 표시: 정상
- 히스토리 로드 + 대화 메시지 복원: 정상
- 사용자 메시지 낙관적 추가: 정상
- SSE 스트리밍 응답: CS와 동일한 baseURL 이슈로 MSW 미매칭 (기존 CS도 동일)
- 뒤로가기/닫기 버튼: 정상
